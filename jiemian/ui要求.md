物料检测模块 - UI需求文档 (最终版)

模块概述

功能定位：上料位物料质量检查与绑定点，通过AGV摄像头检测物料表面是否有划痕，检测合格后将物料信息绑定到当前AGV，并开始生产任务。

核心流程

1. AGV到达上料位，夹取物料
2. 触发物料检测（划痕检测）
3. 检测合格后，将物料信息（型号）绑定到当前AGV
4. 创建生产任务，AGV开始执行任务
5. 检测不合格则需处理物料，重新检测

字段说明：

• 状态指示器：

  • ● 就绪（绿色）：相机准备就绪，等待物料

  • ● 检测中（黄色）：正在进行检测

  • ✅ 合格（绿色）：检测通过，物料无划痕

  • ❌ 不合格（红色）：检测失败，物料有划痕

  • ⚠ 异常（橙色）：相机故障或通信错误

• 当前AGV：显示正在上料位执行操作的AGV编号，括号内显示状态：

  • (等待中)：AGV就绪，等待指令

  • (夹取物料中)：正在夹取物料

  • (检测中)：正在检测

  • (已绑定)：检测合格，已绑定物料

  • (离线)：AGV通信中断

• 预期型号：显示当前任务要求的型号（从任务系统获取）

• 检测结果：

  • ✅ 合格：无划痕，可进入产线

  • ❌ 不合格：有划痕，需更换物料

  • --：检测中或未开始

• 绑定状态：

  • 未绑定：AGV未携带任何物料

  • 检测中：正在检测，结果未出

  • 已绑定(标准版)：检测合格，物料已绑定

  • 绑定失败：检测不合格，未绑定

• 检测时间：最近一次检测完成时间

• 查看检测历史按钮：点击展开/收起历史记录列表

交互逻辑

1. 完整绑定流程


1. AGV到达上料位 → 状态显示当前AGV编号
2. AGV夹取物料 → 状态变"检测中"
3. 开始检测 → 显示"检测中..."
4. 检测合格 → 显示"合格"，自动创建任务
5. 绑定物料 → 状态变"已绑定(型号)"
6. 记录绑定时间 → 更新绑定时间
7. AGV离开 → 状态复位为"就绪"


2. 绑定失败处理流程


1. 检测不合格 → 显示"不合格"，原因
2. 绑定失败 → 状态变"绑定失败"
3. AGV释放物料 → 物料返回物料区
4. 操作员处理 → 选择操作：
   - [重新检测]：物料不动，重新拍照检测
   - [强制通过]：人工确认合格，继续绑定
   - [更换物料]：AGV放下当前物料，取新物料

异常处理

1. 绑定超时


状态: ⚠ 异常
当前AGV: #2
预期型号: [Pro版]
检测结果: ✅ 合格
绑定状态: 绑定超时
原因: 任务创建超时，请检查任务系统
建议: 检查网络连接，重试绑定

[重试绑定] [手动创建任务]


2. AGV通信中断


状态: ⚠ 异常
当前AGV: #2 (离线)
预期型号: [Pro版]
检测结果: ✅ 合格
绑定状态: 通信中断
原因: AGV#2通信中断
建议: 检查AGV连接，恢复后继续绑定

[检查AGV状态] [取消绑定]


3. 相机故障


状态: ⚠ 异常
当前AGV: #2
预期型号: [Pro版]
检测结果: 无法检测
绑定状态: 设备故障
原因: 上料位相机#1无响应
建议: 检查相机连接，重启设备

[重启相机] [手动输入]


权限控制

• 操作员：查看状态，可执行"重新检测"

• 班组长：可执行"强制通过"，需填写原因

• 调度员：可手动创建任务，重新绑定

• 管理员：可配置绑定参数，查看详细日志

与其他模块集成

1. 与AGV监控模块

• 绑定成功后，AGV状态更新为"携带[型号]物料"

• AGV位置信息与绑定信息同步

• 绑定失败时，AGV状态显示"绑定失败"

2. 与任务列表模块

• 绑定成功后自动创建对应任务

• 任务详情中显示绑定AGV和物料信息

• 绑定失败时，任务创建失败，记录原因

3. 与地图模块

• 绑定成功的AGV在地图上显示物料图标

• 不合格物料在地图上标记警示

• AGV与物料的绑定状态实时更新

4. 与系统日志

• 所有绑定操作记录到审计日志

• 绑定失败记录详细错误信息

• 检测结果记录到质量日志

性能要求

1. 响应时间

• 状态更新延迟：< 500ms

• 检测结果响应：< 2s

• 绑定操作完成：< 3s

• 历史记录加载：< 1s

2. 并发能力

• 支持同时处理2-3个AGV的检测请求

• 历史记录支持1000条快速查询

• WebSocket连接稳定，支持断线重连

3. 数据存储

• 检测记录保存30天

• 绑定日志永久保存

• 支持数据备份和恢复
