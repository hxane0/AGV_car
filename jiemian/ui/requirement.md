# 智能物料搬运系统 – 前后端与数据库需求说明

## 1. 项目概述

本项目旨在构建一套面向新能源汽车电池包生产场景的智能物料搬运系统，核心设备为搭载机械臂的AGV（自动导引车）。系统需具备自主导航、精准搬运、物料感知、任务调度、状态监控及人机协同能力。本文档聚焦于**上位机监控系统、任务调度服务、数据存储及与底层设备通信接口**的软件需求，为前后端与数据库的开发提供明确依据。

## 2. 系统角色与用户

- **操作员**：通过监控界面查看系统状态、下达任务、干预异常。
- **维护人员**：查看历史数据、系统日志，进行配置管理。
- **访客/学员**：观摩演示，可通过自然语言与系统交互（需OpenClaw支持）。

## 3. 技术栈建议

| 层次       | 技术选型                                                     |
| ---------- | ------------------------------------------------------------ |
| 前端       | Vue.js / React + TypeScript + ECharts / D3（可视化） + WebSocket |
| 后端       | Python（FastAPI / Django）或 Node.js（Express），与ROS通信使用`roslibpy`或自定义Bridge |
| 数据库     | PostgreSQL / MySQL + Redis（缓存状态）                       |
| 通信协议   | RESTful API（管理类）、WebSocket（实时状态）、ROS Topics/Services（与AGV） |
| 部署环境   | Ubuntu 22.04（与RDK X5兼容），后端服务可容器化               |

## 4. 功能需求

### 4.1 监控与可视化模块（前端核心）

| 功能点             | 描述                                                         |
| ------------------ | ------------------------------------------------------------ |
| **动态地图展示**   | 加载场地二维地图，实时显示AGV位置、航向、规划路径与实际轨迹。支持缩放、平移。 |
| **AGV状态看板**    | 展示每台AGV的电池电量、运行模式（自动/手动）、当前任务、速度、通信质量。 |
| **任务队列管理**   | 以列表或甘特图形式展示所有任务（待执行、执行中、已完成），支持拖拽调整优先级、取消任务。 |
| **视觉检测结果窗** | 实时推送物料抓拍图像，标注缺陷位置、物料分类标签及置信度。   |
| **数据仪表盘**     | 显示累计行驶里程、任务完成数、能耗统计、异常事件次数等。     |
| **多AGV切换**      | 支持在多台AGV间切换监控视角，或全局视图展示所有AGV。        |
| **自然语言交互**   | 集成对话窗口，接收用户语音/文本指令，显示OpenClaw解析结果与执行反馈。 |

### 4.2 任务调度与管理模块（后端核心）

| 功能点               | 描述                                                         |
| -------------------- | ------------------------------------------------------------ |
| **任务创建与下发**   | 操作员可通过界面创建搬运任务（起点、终点、物料类型），后端生成任务并加入全局队列。 |
| **单机任务优化**     | 对单个AGV的多点任务，采用遗传算法等求解最优路径顺序（TSP模型）。 |
| **多机协同调度**     | 集中式任务分配器根据AGV位置、电量、负载动态分配任务，避免冲突。支持基于时间窗的交通管制。 |
| **动态任务插入**     | 允许插入紧急任务，系统自动调整当前任务队列，重新规划。       |
| **任务历史记录**     | 存储所有任务日志，包括执行时间、AGV编号、物料信息、结果状态。 |
| **订单预测（预留）** | 利用历史数据预测高峰期，提前部署AGV（未来可扩展）。         |

### 4.3 数据记录与查询模块

| 功能点             | 描述                                                         |
| ------------------ | ------------------------------------------------------------ |
| **AGV运行日志**    | 记录AGV位置、速度、状态等时序数据，支持回放分析。            |
| **物料检测记录**   | 存储每次物料检测的图像、识别结果（颜色、材质、表面质量）、称重数据。 |
| **异常事件记录**   | 记录系统故障（如路径偏离、通信中断、抓取失败），包括时间、类型、处理方式。 |
| **用户操作日志**   | 记录操作员的所有关键操作（登录、任务下发、参数修改），用于审计。 |

### 4.4 用户与权限管理

| 功能点         | 描述                                                         |
| -------------- | ------------------------------------------------------------ |
| **用户登录**   | 支持用户名/密码认证，区分操作员、管理员等角色。              |
| **权限控制**   | 不同角色可访问的功能不同（如管理员可修改配置，操作员只能执行任务）。 |
| **会话管理**   | 保持登录状态，支持Token鉴权。                                |

## 5. 数据需求（数据库设计初步）

### 5.1 主要实体关系

- **AGV**：记录每台AGV的基本信息与实时状态。
- **任务（Task）**：记录每一次搬运任务。
- **物料检测记录（MaterialInspection）**：关联任务，保存检测结果。
- **异常事件（EventLog）**：记录系统异常。
- **用户（User）**：系统用户。

## 6. 接口需求

### 6.1 前端与后端接口（REST + WebSocket）

- **REST API**（管理类操作）：
  - `GET /api/agv` – 获取所有AGV列表及最新状态
  - `GET /api/agv/{id}` – 获取单个AGV详细信息
  - `POST /api/tasks` – 创建新任务
  - `GET /api/tasks` – 查询任务列表（支持分页、状态过滤）
  - `PUT /api/tasks/{id}/cancel` – 取消任务
  - `GET /api/events` – 查询事件日志
  - `POST /api/login` – 用户登录
  - ……

- **WebSocket**（实时推送）：
  - 订阅 `/topic/agv/status` – 接收所有AGV实时状态更新（位置、电量等）
  - 订阅 `/topic/task/updates` – 接收任务状态变更（新建、开始、结束）
  - 订阅 `/topic/inspection` – 接收物料检测结果图像和数据
  - 订阅 `/topic/logs` – 接收系统日志/事件

### 6.2 后端与AGV/OpenClaw接口

- **与AGV通信**：通过ROS（使用`roslibpy`或自定义Bridge节点）：
  - 订阅AGV的`/odom`、`/battery`、`/amcl_pose`等话题获取状态。
  - 向AGV发布导航目标点（`/move_base_simple/goal`）或取消指令。
  - 调用ROS服务（如`/gripper_control`）控制机械臂动作。

- **与OpenClaw智能体通信**：
  - OpenClaw部署在RDK X5上，通过ROS Bridge与后端交互。
  - 后端接收OpenClaw输出的高层指令（如“去A工位取红色物料”），转换为具体任务。
  - 后端向OpenClaw提供AGV状态、地图信息等，供其决策。

### 6.3 数据采集与存储接口

- 后端需定时（如每秒）从ROS话题获取AGV位置并存入数据库（可选，也可仅保留最新值）。
- 物料检测结果（图像、数据）需通过ROS话题或HTTP上传至后端，并存储。

## 7. 非功能需求

### 7.1 性能
- **实时性**：AGV状态更新延迟 ≤ 200ms，任务指令下发到AGV执行 ≤ 500ms。
- **并发**：支持至少5台AGV同时在线，20个Web客户端同时监控。
- **数据存储**：历史数据保存至少3个月，支持按时间范围快速查询。

### 7.2 可靠性
- **系统可用性**：≥ 99.5%（计划内维护除外）。
- **故障恢复**：后端服务崩溃后自动重启，未完成任务不丢失。
- **数据备份**：每日自动备份数据库。

### 7.3 安全性
- **访问控制**：未授权用户无法访问任何API。
- **通信加密**：所有HTTP和WebSocket通信使用TLS（生产环境）。
- **操作审计**：记录所有关键操作，便于追溯。

### 7.4 可扩展性
- **模块化**：任务调度算法、路径规划算法可插拔替换。
- **多AGV支持**：新增AGV只需在数据库添加记录，无需修改代码。
- **前端插件化**：新增监控组件不影响核心框架。

### 7.5 可维护性
- **日志**：后端输出结构化日志（JSON格式），便于ELK等工具收集分析。
- **配置管理**：所有配置（如ROS话题名、数据库连接）通过环境变量或配置文件管理。

## 8. 其他说明

- 系统需提供**模拟调试模式**，在没有真实AGV时可模拟数据流，便于前端开发和测试。
- 前端界面应支持**中英文切换**（根据报告场景，可能用于教学演示）。
- 数据库初始数据需包含场地地图的坐标信息、工位定义等。
