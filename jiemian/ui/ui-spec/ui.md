# 智能物料搬运系统 – UI 设计规范

## 1. 整体布局

```
┌─────────────────────────────────────────────────────────────┐
│  顶部导航栏（Logo + 系统名称 + 用户信息 + 语言切换 + 退出）    │
├──────────┬──────────────────────────────────────────────────┤
│          │                                                  │
│  左侧    │              主内容区                            │
│  导航    │                                                  │
│  菜单    │                                                  │
│          │                                                  │
└──────────┴──────────────────────────────────────────────────┘
```

- 顶部导航栏高度：60px
  - 背景：半透明玻璃底 rgba(15, 20, 35, 0.75) + backdrop blur(16px)，页面滚动时内容可透过导航栏隐约可见
  - 底部边缘：1px 半透明描边 rgba(255,255,255,0.06)，模拟玻璃板边缘光
  - Logo / 系统名称：文字 #E0E6ED，无多余装饰
  - 右侧功能区（用户信息、语言切换、退出）：按钮透明底，hover 时出现 rgba(255,255,255,0.06) 圆角底色，150ms 过渡
  - WebSocket 连接指示圆点：带语义色微辉光（box-shadow 0 0 4px）
  - 模拟模式横幅：半透明橙色玻璃底 rgba(250,140,22,0.12) + 橙色文字，取代纯色实底
- 左侧菜单宽度：220px（可折叠至 64px）
  - 背景：半透明玻璃底 rgba(15, 20, 35, 0.6) + backdrop blur(16px)，右侧边缘 1px 半透明描边
  - 菜单项默认态：纯文字 + 图标，无背景
  - 菜单项 Hover 态：出现 rgba(255,255,255,0.04) 圆角底色块，150ms 过渡
  - 菜单项选中态：左侧 2px 宽主色竖条 #4A90D9，底色 rgba(74,144,217,0.08)，文字变为主色
  - 折叠态 tooltip：玻璃底气泡（半透明 + 模糊 + 圆角 8px）
  - 子菜单展开：无位移动画，仅透明度 0→1 渐入，150ms
- 主内容区：自适应剩余宽度
- 页面底层背景色：#0B0F18（深冷黑，微偏蓝）

### 3.1 监控总览（Dashboard）

**布局：** 4 列卡片网格 + 下方图表区

```
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│ AGV在线数 │ 今日任务数 │ 异常事件数 │ 平均电量  │ 累计里程  │
│   5/5    │   128    │    2     │  78%     │ 1024 km  │
└──────────┴──────────┴──────────┴──────────┴──────────┘
┌─────────────────────┬───────────────────────┐
│  任务完成趋势折线图   │   AGV 状态分布饼图     │
│  （近7天）           │                       │
└─────────────────────┴───────────────────────┘
┌─────────────────────┬───────────────────────┐
│  能耗统计柱状图       │   最近异常事件列表     │
└─────────────────────┴───────────────────────┘
```

**数据指标卡片：**
- 图标 + 数值 + 单位 + 与昨日对比（↑↓箭头）
- 颜色：正常绿色，异常红色，警告黄色

**视觉风格（液态玻璃）：**
- 每张指标卡片为独立玻璃卡：半透明背景 rgba(255,255,255,0.04) + backdrop blur(16px) + 1px rgba(255,255,255,0.08) 描边 + 圆角 12px
- 图标使用线性（outline）风格，颜色为对应语义色的降饱和度变体
- 数值字号 28px semi-bold，颜色 #E0E6ED；对比箭头使用低饱和度语义色
- 卡片间距 16px；Hover 时背景透明度微增至 0.07，150ms ease 过渡
- 图表容器：每个图表区域为一张玻璃卡，规格同指标卡片
- 图表内部背景完全透明，让玻璃底透出
- 折线图/柱状图配色使用主色 #4A90D9 及其 20% 透明度填充区域
- 饼图扇区使用低饱和度色系，扇区间留 2px 间隙（深底色露出）
- 坐标轴线/网格线使用 rgba(255,255,255,0.06)
- "最近异常事件列表"嵌在玻璃卡内，行 hover 使用 rgba(255,255,255,0.03) 底色

### 3.2 地图监控（Map View）

**定位：** 由二维平面图升级为**三维可实时交互**的监控场景；**不再提供**右侧 AGV 列表面板。AGV 状态与任务摘要通过场景内 **Hover 悬浮卡** 获取，完整信息通过 **AGV 详情页** 获取。

**布局：** 主内容区为**全宽三维视口**；可选在角落放置极简辅助控件（如重置视角、适合全部 AGV），不占用独立侧栏。

```
┌────────────────────────────────────────────────────┐
│  三维场景视口（轨道旋转 / 缩放 / 平移）              │
│                                                    │
│   [带厚度的地图载体平板]                             │
│        [AGV 立方体] …  [AGV 立方体]                  │
│        （规划路径：按车开关，3D 折线略高于表面）        │
│                                                    │
│  （可选）左下角：玻璃风小工具条 — 重置视角 / 全局取景   │
└────────────────────────────────────────────────────┘
```

**组件结构（逻辑分层，实现时可映射为同一视口中的子模块）：**

| 模块 | 职责 |
|------|------|
| **Scene** | 场景根：世界坐标系、环境光、相机绑定；聚合 MapPlate、全部 AGVEntity、PathLayer；与后端地图/定位坐标约定对齐。 |
| **MapPlate（地图载体）** | 默认：**带厚度的平板**（简单盒状几何即可），表示场地承载面；**预留**通过资源标识替换为自定义三维模型（具体字段见 `api-spec`）。 |
| **AGVEntity** | 单车实例：**业务 ID 全局唯一**且与后端 AGV `id` 一致；默认外观为**立方体**；位置与朝向随实时数据更新；**预留**替换为真实车体模型。 |
| **PathLayer** | **规划路径**：在载体表面上方绘制的 **3D 折线**（略高于表面，避免与面相交闪烁）；**按 AGV 独立**控制是否渲染（见交互）。 |
| **ViewTooltip（悬浮信息卡）** | Hover 时展示的简报层（非场景内玻璃 mesh）：电量、状态、当前任务摘要等；定位跟随光标或该车屏幕投影锚点。 |

**相机：** 支持**绕目标轨道旋转**、**缩放**、**平移**（平移沿合理约束，以保证始终能观察地图载体与小车）。

**场景元素说明：**
- **AGV 状态色**：与 4.1 语义一致（运行绿、待机青、充电橙、故障红、离线灰）；可在立方体顶面或棱线做弱对比，**不使用**复杂发光或持续动画。
- **工位（可选）**：可在载体平面上以简化标注（框线、立柱或名称标签）表示，避免遮挡小车主体。
- **实际历史轨迹**：本页**不强制**绘制；若需与详情页「历史轨迹回放」能力对齐，可在后续迭代中单开开关，本规范不扩展复杂轨迹系统。

**交互：**
- **Hover 小车**：显示 **ViewTooltip**（电量、状态徽章、当前任务摘要；电量条规格复用 4.2）。
- **左键点击小车**：跳转至该车的 **AGV 详情**页（沿用系统内 AGV 详情路由约定）。
- **右键点击小车**：**切换**该车在 **PathLayer** 中规划路径的 **显示 / 隐藏**；各车相互独立，可同时多车开启。
- **多车实时**：视口内可同时存在多辆在线 AGV；位置与状态通过 **WebSocket 推送**（推荐）或 **REST 轮询** 刷新，频率与断线策略见 `api-spec`。

**规划路径（视觉）：**
- 三维折线，整体悬浮于地图载体上表面之上；线色建议主色 #4A90D9 或与深色背景对比清晰的单色；**无**虚线/粒子等复杂效果亦可接受。
- **默认不显示**；仅当用户对该车执行右键打开后，才渲染该车的规划路径。

**视觉风格：**
- **场景内几何体与地图载体**：**不使用**玻璃模糊、半透明材质叠层，保证清晰可辨。
- **ViewTooltip**、**可选角落工具条**：可采用液态玻璃（半透明底 + 模糊 + 细描边 + 圆角，参见 4.5、4.6 Tooltip）。
- **动效**：禁止复杂动画；仅允许全局 4.5 允许的短过渡（如 Tooltip 显隐 ≤200ms）。

**与 AGV 管理的关系：** 本页**不替代**「AGV 列表」；列表页仍为表格化检索与批量管理入口。

### 3.4 AGV 详情页

根据您提供的项目需求分析报告，我将为您整理AGV小车的部件划分及对应信息，并修改详情页设计需求。
## 一、AGV部件划分及信息
根据报告中的硬件约束和系统集成方案，将AGV小车分为以下**5大核心部件**：
### 1. 移动底盘（CompassQ2）
- **型号/规格**：359×313.5×114mm，重量6.7kg
- **驱动系统**：四轮独立驱动，QMA-10全向轮
- **运动性能**：最大速度0.65m/s，最大自转速度140°/s，最小转弯半径0m
- **控制器**：RHF407开发板（STM32核心）
- **电源系统**：12V/10.4Ah锂离子电池，典型续航约10h
- **悬挂系统**：共轴摆式悬挂
### 2. 协作机械臂（自研）
- **型号/规格**：总重量>6kg，底座安装尺寸≤300mm×250mm
- **自由度**：三自由度（带水平回转关节）
- **末端执行器**：真空吸盘（独立气泵+电磁阀控制）
- **负载能力**：最大抓取重量≥200g
- **控制器**：STM32微控制器
### 3. 多模态感知系统
- **激光雷达**：N10P，用于360°环境扫描、SLAM建图、避障
- **双目深度相机**：RGB+深度信息，用于物料识别、位姿定位、质检
- **超声传感器**：3路ULB-1，用于近距离避障
- **IMU惯性单元**：用于姿态测量与航迹推算
- **编码器**：电机编码器，用于里程计
- **麦克风阵列**：M260C，用于语音交互与声学监测
### 4. 边缘计算平台（RDK X5）
- **算力**：10 TOPS
- **CPU**：8核A55
- **操作系统**：Ubuntu 22.04
- **功能**：运行OpenClaw智能体框架、本地大模型（Ollama部署）、自然语言处理
### 5. 执行与辅助系统
- **电池管理**：12V/10.4Ah锂电池，实时电量监测
- **通信系统**：CAN总线、RS232、以太网/WiFi
- **安全模块**：急停模块、安全光栅、限位装置
- **结构支撑**：安装支架、物料托盘、电源降压模块

### 3.4 AGV 详情页（交互式3D展示版）
**整体布局：** 左右分栏 + 浮动信息卡片
```
┌────────────────────────────────┬──────────────────────────────────┐
│                                │  基本信息卡（玻璃卡）              │
│                                │  编号/型号/当前任务状态            │
│                                ├──────────────────────────────────┤
│   AGV 3D模型展示区              │  实时状态卡（玻璃卡）              │
│   - 可旋转、缩放查看             │  电量/速度/位置/朝向/模式          │
│   - 点击部件高亮显示             │  通信质量/运行时间                 │
│   - 悬停显示部件名称             ├──────────────────────────────────┤
│   - 支持视角重置按钮             │  历史轨迹回放（小地图 + 时间轴）   │
│                                ├──────────────────────────────────┤
│                                │  任务历史列表（分页表格）           │
│                                │  （可上下滚动查看）                │
└────────────────────────────────┴──────────────────────────────────┘
部件信息浮动卡片（点击部件后显示，可拖动）
┌──────────────────────────────┐
│  部件名称            [×]     │
├──────────────────────────────┤
│  型号/规格                   │
│  核心参数列表                 │
│  当前状态                     │
│  健康度/告警信息              │
└──────────────────────────────┘
```
**视觉风格（液态玻璃）：**
- 整体背景：深色调半透明玻璃效果（backdrop-blur: 20px）
- 左侧3D模型区：
  - 背景：渐变玻璃容器，内阴影 + 微光效果
  - 模型：高光材质，点击部件时发光高亮（主色光晕）
  - 悬停：部件轮廓高亮 + 鼠标变为手型
  - 视角控制按钮：玻璃质感，带微投影
- 右侧信息区：
  - 基本信息卡 & 实时状态卡：并排两张玻璃卡，半透明背景 + 模糊 + 描边 + 圆角 12px
  - 标签文字使用次要色 #7A8494，数值使用主文字色 #E0E6ED
  - 动态数值更新时以透明度脉冲提示变化（opacity 0.5→1.0，180ms ease-out）
- 历史轨迹回放：玻璃卡容器；小地图保持清晰
- 任务历史表格：复用列表页玻璃表格规格
**部件信息浮动卡片设计：**
- 外观：玻璃卡片样式（半透明 + 模糊 + 描边 + 圆角 8px）
- 尺寸：宽度 280px，高度自适应（最小 200px）
- 位置：初始出现在右侧区域中央，可拖动调整位置
- 标题栏：
  - 部件名称（加粗） + 关闭按钮（右上角 ×）
  - 鼠标悬停在标题栏时显示拖动提示
- 内容区：
  - 分段显示：基本信息、核心参数、当前状态、健康指标
  - 参数值使用高亮色显示
  - 状态图标（正常/告警/故障）
- 交互：
  - 左键按住标题栏可拖动
  - 点击关闭按钮消失
  - 拖动时有微光阴影效果
  - 无法拖出右侧区域边界
**部件对应信息映射：**
| 点击部件 | 显示信息 |
|---------|---------|
| 底盘 | 型号、尺寸、驱动方式、最大速度、电池容量、续航时间 |
| 机械臂 | 自由度、负载能力、末端执行器、精度范围、工作状态 |
| 激光雷达 | 型号、扫描范围、分辨率、建图状态、障碍物检测 |
| 深度相机 | 型号、分辨率、视野范围、识别模式、当前任务 |
| 麦克风阵列 | 型号、拾音范围、降噪状态、语音识别状态 |
| 计算平台 | 型号、算力、CPU使用率、内存占用、温度 |
| 电池 | 电压、容量、剩余电量、健康度、充放电次数 |
**交互逻辑：**
1. 进入详情页：3D模型默认正面视角，右侧显示基本信息
2. 鼠标悬停部件：部件高亮 + 显示部件名称提示
3. 点击部件：右侧弹出该部件详细信息卡片，可拖动
4. 再次点击同一部件：信息卡片消失
5. 点击另一个部件：信息卡片更新为新部件信息，位置重置
6. 拖动模型：旋转、缩放不影响信息卡片位置
7. 关闭信息卡片：点击右上角 × 或点击空白区域
**性能要求：**
- 3D模型渲染：60fps 流畅度
- 信息卡片拖动：流畅无卡顿
- 状态数据更新：实时推送，延迟 < 1s
- 部件点击响应：< 200ms
**扩展功能（可选）：**
- 支持多部件同时显示信息卡片（但建议最多3个，避免遮挡）
- 部件健康状态在3D模型上用颜色标识（绿色正常、黄色告警、红色故障）
- 支持部件维保记录查询链接
这样的设计既保留了原有的监控信息展示，又增加了直观的3D交互体验，让用户能够深入了解AGV各部件的详细信息，符合现代工业监控系统的交互趋势。

|------|--------|--------|

| 故障/失败/取消 | #f5222d | rgba(245,34,45,0.12) |
| 离线 | #8c8c8c | rgba(140,140,140,0.12) |

### 4.2 电量显示

- 进度条高度：4px，全圆角，底色 rgba(255,255,255,0.06)
- 填充部分顶部有 1px 高光线增添光泽感
- \> 60%：绿色进度条
- 20%–60%：黄色进度条
- < 20%：红色进度条 + 旁边静态红色三角警告图标（不闪烁）

### 4.3 实时数据更新

- WebSocket 连接状态指示器（右上角小圆点：绿=已连接，红=断开），带 0 0 4px 语义色辉光
- 数据更新时，对应数值以透明度脉冲提示变化：opacity 0.5→1.0，180ms ease-out

### 4.4 响应式断点

| 断点 | 宽度 | 说明 |
|------|------|------|
| 桌面 | ≥ 1440px | 完整布局 |
| 笔记本 | 1024–1439px | 左侧菜单可折叠 |
| 平板 | 768–1023px | 左侧菜单默认折叠 |

### 4.5 通用玻璃材质规范

**卡片/面板基础样式：**
- 背景：rgba(255,255,255,0.04) + backdrop-filter: blur(16px)
- 边框：1px solid rgba(255,255,255,0.08)
- 圆角：大卡片 12px / 小元素 8px / 登录卡 16px
- 投影：0 4px 24px rgba(0,0,0,0.3)（深色柔和投影，营造浮空感）
- 浮动层（弹窗/气泡）：背景 rgba(255,255,255,0.08)，更高透光度

**动效约束（全局）：**
- 所有过渡动效时间严格 ≤ 200ms，推荐 120–180ms
- 仅允许微交互：hover / focus 态变化
- Hover 态：背景透明度轻微提升（0.04→0.07），150ms ease
- Focus 态：边框高光增强为 rgba(74,144,217,0.4)，120ms
- 禁止一切弹跳、缩放、位移超过 4px 的动画

### 4.6 通用组件风格

**表格通用：**
- 表格容器为玻璃卡
- 表头固定，rgba(255,255,255,0.04) 微底色区分，文字 #7A8494
- 不使用斑马纹，改为 hover 高亮行 rgba(255,255,255,0.03)

- 分页器按钮使用玻璃底小方块，选中页码为主色底

**输入框通用：**

**弹窗/模态框：**
