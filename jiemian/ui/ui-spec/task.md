任务管理监控界面 - UI需求文档

1. 界面定位

本界面为“蓝牙耳机智能生产线监控系统”的独立任务管理终端，专注于生产订单的创建、查看、调度与状态监控。界面与AGV状态界面、产线地图界面数据实时同步，但功能高度聚焦于任务流本身。

2. 整体布局

采用单页应用程序(SPA)设计，布局清晰分为上下两大区域，无复杂侧边栏。

--------------------------------------------
| [状态栏] 总任务: 8 (3执行/4等待/1完成)   |
--------------------------------------------
|           任务创建面板                    |
--------------------------------------------
|           任务列表主面板                  |
| (占据界面70%以上空间)                    |
--------------------------------------------
|           运行日志面板 (可折叠)           |
--------------------------------------------


3. 核心功能模块详述

3.1 全局状态栏 (顶部)

目标：提供任务维度的宏观概览。
• 任务统计概览：

  总任务: 8 执行中: 3 等待中: 4 已完成: 1 异常: 0
• 产品型号分布（今日）：

  标准版: 5 Pro版: 2
 无线充新版: 1

3.2 任务创建面板

目标：快速下达新的生产指令。
• 紧凑型创建表单：

  [创建新任务]
  ┌─────────────────────────────────────┐
  │ 产品型号: [标准版 ▼]                 │
  │ 数量:    [1]   (1-99)               │
  │ 优先级:  [● 普通] [○ 紧急]           │
  │                                     │
  │        [确认创建]                    │
  └─────────────────────────────────────┘
  
  • 产品型号选择：决定生产路径的核心参数。

  • 优先级：紧急任务将被系统优先调度。

  • 点击确认创建后，任务立即出现在下方列表的“等待”区顶部。

3.3 任务列表主面板 (核心)

目标：全面展示和管理所有任务的生命周期。
• 列表表头：

  任务ID 产品型号 状态 进度 执行AGV 操作

• 任务行详情（每行示例）：

  | T-010 | Pro版    | ▶ 执行中 | 2/5 | #2     | [暂停] [取消] |
  | T-011 | 标准版   | ● 等待   | 0/3 | --     | [开始] [取消] |
  | T-009 | 无线充新版| ✓ 完成  | 5/5 | #1     | [详情]        |
  
  • 状态列：使用图标+文字清晰标示。

    ◦ ● 等待 (灰色)

    ◦ ▶ 执行中 (绿色)

    ◦ ❙❙ 暂停 (黄色)

    ◦ ✓ 完成 (蓝色)

    ◦ ✕ 取消/⚠ 异常 (红色)

  • 进度列：显示为 已完成数/总数量。执行中任务可有细进度条动画。

  • 执行AGV列：显示执行此任务的AGV编号。若为“--”表示未分配。

  • 操作列：状态依赖的按钮组，确保逻辑严谨。

    ◦ 等待任务：[开始] [取消]

    ◦ 执行中任务：[暂停] [取消]

    ◦ 暂停任务：[继续] [取消]

    ◦ 完成/取消任务：仅[详情]按钮，查看历史记录。

• 列表功能：

  • 筛选器：在列表上方提供快速筛选按钮：[全部] [等待] [执行] [完成] [异常]。

  • 排序：点击表头可按任务ID、状态、优先级排序。

  • 批量操作（可选）：勾选多个等待状态任务，可进行批量开始或批量取消。

3.4 任务详情视图 (模态框/侧滑)

目标：查看单个任务的完整时间线和详情。
• 触发：点击已完成或取消任务的[详情]按钮，或双击任何任务行。

• 显示内容：

  • 基础信息：任务ID、产品型号、创建时间、完成/取消时间、总耗时。

  • 物料流：显示该任务下每个物料的当前状态（如“已上线”、“在工位2加工”、“已完成下线”）。

  • 事件时间线：以时间轴形式展示关键日志，例如：

    14:20:01 任务创建
    14:22:15 分配给 AGV#2
    14:23:30 AGV#2 在上料位取料成功
    14:25:00 第一个单元在工位1完成装配
    14:30:12 所有5个单元完成下线，任务结束
    

3.5 运行日志面板 (底部，默认展开)

目标：记录与任务相关的所有系统事件，便于追踪和排错。
• 日志流：持续滚动显示最新日志。

• 关键信息类型（需记录）：

  • 任务生命周期：任务T-010（Pro版）已创建。 任务T-010已开始执行。 任务T-010全部完成。

  • 资源调度：任务T-010已分配给AGV#2。

  • 流程节点：任务T-010（由AGV#2执行）的第一个物料已到达工位2。 任务T-010（Pro版）的物料已跳过工位3。

  • 异常：任务T-010在工位2发生超时，已暂停。

• 筛选：可筛选[全部]、[任务事件]、[警告/错误]。

4. 数据联动说明

• 当任务被开始时，系统后台自动调度AGV，本界面“执行AGV”列将更新为具体的AGV编号。

• 当AGV在其他界面被手动干预（如取消任务），本界面任务状态将自动同步更新为取消或暂停。

• 本界面是“生产计划”的输入和体现点，与体现“实时位置”的地图界面、体现“设备健康”的AGV状态界面互为补充。

此独立任务管理界面功能完整、聚焦，能够高效支撑生产订单的下达与全程跟踪，开发复杂度可控。
