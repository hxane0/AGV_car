系统模块 - 事件日志部分 UI需求文档

模块概述

功能定位：实时记录系统运行时自动产生的各种事件，用于监控系统运行状态、排查问题、追踪业务流程。

设计原则

1. 实时性：新事件实时显示，无需手动刷新
2. 可读性：事件描述清晰，包含所有关键信息
3. 可筛选：支持按级别、时间、模块等多维度筛选
4. 简洁高效：不干扰主流程，但能快速查看关键信息

3. 筛选与搜索功能

3.1 级别筛选按钮

• [全部]：显示所有级别的事件

• [信息]（蓝色）：正常系统事件，如任务创建、完成等

• [警告]（黄色）：需要注意但非致命的问题，如等待超时

• [错误]（红色）：系统错误，需要立即关注，如设备故障

3.2 高级筛选（点击展开）


高级筛选:
模块: [全部 ▼]          时间: [今天 ▼]
AGV: [全部 ▼]          任务: [全部 ▼]
   [应用筛选] [重置]


筛选选项：
• 模块：全部、任务系统、AGV调度、物料检测、工位控制、系统

• 时间：今天、最近1小时、最近24小时、自定义

• AGV：全部、#1、#2、#3、#4

• 任务：全部、或输入任务ID

3.3 搜索功能

• 支持关键词搜索，在所有事件内容中匹配

• 支持模糊搜索

• 搜索结果高亮显示

4. 事件级别定义

4.1 信息级别（蓝色）

• 系统启动/关闭

• 任务创建/开始/完成

• AGV状态变化（空闲→执行任务，开始充电等）

• 物料检测结果

• 工位加工完成

• 用户登录/登出

示例：

14:30:15 [信息] 任务T-015(Pro版)已创建，分配给AGV#2。
14:28:40 [信息] AGV#3到达下线点，任务T-014完成。


4.2 警告级别（黄色）

• AGV等待超时

• 物料识别置信度过低

• AGV电量低于20%

• 工位加工时间过长

• 网络延迟较高

• 缓冲区接近满载

示例：

14:25:12 [警告] AGV#2在工位3等待超时(超过2分钟)。
14:22:30 [警告] AGV#1电量低(18%)，建议充电。


4.3 错误级别（红色）

• 设备通信中断

• 任务执行失败

• AGV定位丢失

• 数据库连接失败

• 关键服务异常

• 安全门异常打开

示例：

14:18:12 [错误] 上料位相机通信中断，已尝试重连。
14:15:20 [错误] 任务T-010执行失败，原因：机械臂抓取失败。


5. 界面控制功能

5.1 控制按钮

• [自动滚动]：默认开启，新事件自动滚动到视图底部

• [暂停]：暂停自动滚动，便于查看历史事件

• [清空]：清空当前显示的事件列表（不删除数据库记录）

• [导出]：将当前筛选结果导出为CSV/TXT文件

• [设置]：进入事件日志设置

5.2 显示选项

7. 关键事件类型

7.1 任务相关事件


任务创建 → 任务分配 → 任务开始 → 任务进度更新 → 任务完成/失败


事件链示例：

14:30:15 [信息] 创建任务T-015，型号:Pro版，数量:5。
14:30:16 [信息] 任务T-015分配给AGV#2。
14:30:20 [信息] AGV#2开始执行任务T-015。
14:31:05 [信息] 任务T-015进度:1/5完成。
14:35:12 [信息] 任务T-015全部完成，耗时4分52秒。


7.2 AGV相关事件


AGV状态变化 → AGV位置更新 → AGV电量变化 → AGV任务执行


事件链示例：

14:30:20 [信息] AGV#2状态:空闲 → 执行任务T-015。
14:30:25 [信息] AGV#2位置:上料位 → 前往工位1。
14:30:40 [信息] AGV#2到达工位1。
14:31:00 [信息] AGV#2电量:85% → 84%。


7.3 物料检测事件


物料到达 → 开始检测 → 检测结果 → 绑定结果


事件链示例：

14:23:00 [信息] 上料位检测到物料。
14:23:01 [信息] 开始物料检测(AGV#2)。
14:23:05 [信息] 检测结果:合格，型号:Pro版。
14:23:06 [信息] 物料绑定成功:Pro版 → AGV#2。


8. 特殊事件处理

8.1 批量操作事件

当执行批量操作时，合并显示：

14:30:00 [信息] 批量创建3个任务: T-015,T-016,T-017。


点击可展开详情：

任务详情:
• T-015: Pro版, 5个, 分配给AGV#2
• T-016: 标准版, 3个, 分配给AGV#3
• T-017: 无线充新版, 2个, 分配给AGV#1


8.2 事件关联

相关事件可链接查看：

14:25:12 [警告] AGV#2在工位3等待超时。

9. 视觉设计

9.1 颜色规范

• 信息级别：蓝色 (#1890ff)，图标: ℹ

• 警告级别：黄色 (#faad14)，图标: ⚠

• 错误级别：红色 (#f5222d)，图标: ✕

• 调试级别：灰色 (#bfbfbf)，图标: 🐛

9.2 布局尺寸

• 收起状态：300px × 40px（只显示标题栏）

• 展开状态：300px × 400px（默认）

• 最大化：600px × 600px（双击标题栏最大化）

9.3 交互效果

• 新事件到达：轻微背景闪烁

• 错误事件：红色边框闪烁3次

• 警告事件：黄色边框闪烁1次

• 鼠标悬停：背景色变浅

• 双击事件：展开详情模态框

10. 数据存储

10.1 数据库结构

CREATE TABLE event_logs (
  id BIGINT PRIMARY KEY,
  timestamp DATETIME NOT NULL,
  level VARCHAR(10) NOT NULL, -- INFO, WARN, ERROR, DEBUG
  module VARCHAR(50) NOT NULL, -- 模块名称
  event_code VARCHAR(100), -- 事件代码
  message TEXT NOT NULL, -- 事件描述
  details JSON, -- 详细数据
  related_agv VARCHAR(20), -- 相关AGV
  related_task VARCHAR(20), -- 相关任务
  user_id VARCHAR(50), -- 操作用户
  ip_address VARCHAR(50) -- 来源IP
);


10.2 事件代码示例

• TASK_CREATED：任务创建

• TASK_COMPLETED：任务完成

• AGV_STATE_CHANGED：AGV状态变化

• MATERIAL_INSPECTED：物料检测完成

• AGV_WAIT_TIMEOUT：AGV等待超时

• DEVICE_CONNECTION_LOST：设备连接断开
